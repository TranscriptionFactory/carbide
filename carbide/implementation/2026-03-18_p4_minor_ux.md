# P4 Minor UX — Sprint 2026-03-18

## Items

### #16 Swap keyboard shortcuts: Outline vs OmniFind

**Status:** ✅ Implemented

**Problem:** Outline Panel was bound to `CmdOrCtrl+Shift+O` and Search All Vaults to `CmdOrCtrl+Shift+F`. Users expected the reverse — `Shift+F` for the outline (find structure) and `Shift+O` for omni-search (open/search).

**Changes:**

- `src/lib/features/hotkey/domain/default_hotkeys.ts` — swapped the two key bindings
- `tests/unit/services/hotkey_service.test.ts` — updated assertion to match new default

---

### #17 Remove window zoom entirely

**Status:** ✅ Implemented

**Problem:** Tauri's built-in `Cmd+Plus`/`Cmd+Minus`/`Cmd+0` zoom hotkeys interfere with the app's own text-level zoom or layout. Users accidentally zoom the webview.

**Changes:**

- `src-tauri/tauri.conf.json` — `zoomHotkeysEnabled: false` (main window)
- `src/lib/features/window/adapters/window_tauri_adapter.ts` — `zoomHotkeysEnabled: false` (child windows)

---

### #18 Add filename + date_created to frontmatter

**Status:** ✅ Implemented

**Problem:** New notes start with no frontmatter. Users want `title` and `date_created` auto-populated on creation.

**Changes:**

- `src/lib/features/note/domain/ensure_open_note.ts` — added `format_date_iso()` and `build_initial_frontmatter()` helpers; `create_untitled_open_note()` now generates initial markdown with YAML frontmatter containing `title` (quoted) and `date_created` (ISO 8601 `YYYY-MM-DD`)
- `tests/unit/utils/ensure_open_note.test.ts` — updated markdown assertion to expect frontmatter content

**Frontmatter format:**

```yaml
---
title: "Untitled-1"
date_created: 2026-03-18
---
```
